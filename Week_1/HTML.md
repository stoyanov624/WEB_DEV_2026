# Упражнение 1

## Въведение

HTML (HyperText Markup Language) е езикът за описание на структурата на уеб страниците. Той не е език за програмиране, а **маркиращ (описателен) език** — казва на браузъра _какво_ да покаже, не _как_ да го изчисли. Всяка уеб страница, която виждате в интернет, има HTML в основата си.

---

## 1. Структура на HTML документ

Всеки HTML файл следва една и съща базова структура:

```html
<!DOCTYPE html>
<html lang="bg">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Моята първа страница</title>
  </head>
  <body>
    <!-- Тук отива видимото съдържание -->
    <h1>Здравей, свят!</h1>
    <p>Това е моята първа уеб страница.</p>
  </body>
</html>
```

### Какво означава всеки ред?

| Елемент                      | Предназначение                                       |
| ---------------------------- | ---------------------------------------------------- |
| `<!DOCTYPE html>`            | Казва на браузъра, че документът е HTML5             |
| `<html lang="bg">`           | Коренов елемент; `lang="bg"` указва език — български |
| `<head>`                     | Мета-информация (не се вижда на страницата)          |
| `<meta charset="UTF-8">`     | Кодиране — поддържа кирилица и специални символи     |
| `<meta name="viewport" ...>` | Прави страницата адаптивна за мобилни устройства     |
| `<title>`                    | Заглавие в таба на браузъра                          |
| `<body>`                     | Всичко видимо за потребителя                         |

> 💡 **Запомни:** `<head>` е за браузъра, `<body>` е за потребителя.

## 2. Текстови елементи

### 2.1 Заглавия (Headings)

HTML предлага 6 нива заглавия — от `<h1>` (най-важно) до `<h6>` (най-маловажно):

```html
<h1>Главно заглавие</h1>
<h2>Подзаглавие</h2>
<h3>Раздел</h3>
<h4>Подраздел</h4>
```

> Използвай само едно `<h1>` на страница. Заглавията трябва да следват логическа йерархия (не прескачай от `<h1>` директно към `<h4>`).

### 2.2 Параграфи и форматиране

```html
<p>
  Това е обикновен параграф. Браузърът автоматично добавя разстояние преди и
  след него.
</p>

<p>
  Текст с <strong>удебелен (важен)</strong> и
  <em>наклонен (акцентиран)</em> текст.
</p>

<p>Нов ред в рамките на параграф:<br />Текстът продължава на нов ред.</p>

<hr />
<!-- Хоризонтална линия — разделител между секции -->
```

| Таг        | Значение                   | Визуален резултат  |
| ---------- | -------------------------- | ------------------ |
| `<strong>` | Семантично важен текст     | **удебелен**       |
| `<em>`     | Акцентиран текст           | _наклонен_         |
| `<br>`     | Нов ред (без нов параграф) | —                  |
| `<hr>`     | Тематичен разделител       | хоризонтална линия |

## 3. Списъци

### Неподреден списък (bulleted)

```html
<ul>
  <li>HTML — структура</li>
  <li>CSS — стил</li>
  <li>JavaScript — поведение</li>
</ul>
```

### Подреден списък (numbered)

```html
<ol>
  <li>Отвори текстов редактор</li>
  <li>Напиши HTML кода</li>
  <li>Запази файла с разширение .html</li>
  <li>Отвори го в браузъра</li>
</ol>
```

### Вложени списъци

```html
<ul>
  <li>
    Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>
    Backend
    <ul>
      <li>PHP</li>
      <li>Python</li>
      <li>Node.js</li>
    </ul>
  </li>
</ul>
```

## 4. Връзки (Links)

```html
<!-- Външна връзка (отваря се в нов таб) -->
<a href="https://www.google.com" target="_blank">Отиди в Google</a>

<!-- Вътрешна връзка (към друга страница в сайта) -->
<a href="kontakti.html">Контакти</a>

<!-- Връзка към секция на същата страница -->
<a href="#section2">Към Раздел 2</a>

<!-- Целева секция -->
<h2 id="section2">Раздел 2</h2>

<!-- Линк за имейл -->
<a href="mailto:info@example.com">Пиши ни</a>
```

| Атрибут           | Описание                                 |
| ----------------- | ---------------------------------------- |
| `href`            | Адрес на дестинацията (задължителен)     |
| `target="_blank"` | Отваря в нов таб                         |
| `id`              | Уникален идентификатор за котва (anchor) |

## 5. Изображения

```html
<!-- Основен синтаксис -->
<img src="images/logo.png" alt="Лого на компанията" width="200" />

<!-- Изображение от интернет -->
<img src="https://via.placeholder.com/300x200" alt="Примерно изображение" />

<!-- Изображение като линк -->
<a href="https://example.com">
  <img src="banner.jpg" alt="Кликни тук" />
</a>
```

> **Атрибутът `alt`.** Той описва изображението за хора с нарушено зрение (screen readers) и се показва, ако снимката не се зареди.

## 6. Таблици

```html
<table>
  <thead>
    <tr>
      <th>Име</th>
      <th>Възраст</th>
      <th>Град</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Иван</td>
      <td>22</td>
      <td>София</td>
    </tr>
    <tr>
      <td>Мария</td>
      <td>19</td>
      <td>Пловдив</td>
    </tr>
    <tr>
      <td>Георги</td>
      <td>25</td>
      <td>Варна</td>
    </tr>
  </tbody>
</table>
```

| Таг       | Роля                           |
| --------- | ------------------------------ |
| `<table>` | Контейнер на таблицата         |
| `<thead>` | Заглавна част                  |
| `<tbody>` | Тяло с данни                   |
| `<tr>`    | Ред (table row)                |
| `<th>`    | Клетка-заглавие (table header) |
| `<td>`    | Клетка с данни (table data)    |

## 7. Формуляри

Формулярите са начинът, по който потребителят въвежда данни:

```html
<form action="/submit" method="POST">
  <label for="name">Име:</label>
  <input type="text" id="name" name="name" placeholder="Въведи име" required />

  <label for="email">Имейл:</label>
  <input
    type="email"
    id="email"
    name="email"
    placeholder="example@mail.com"
    required
  />

  <label for="age">Възраст:</label>
  <input type="number" id="age" name="age" min="1" max="120" />

  <label for="gender">Пол:</label>
  <select id="gender" name="gender">
    <option value="">-- Избери --</option>
    <option value="male">Мъж</option>
    <option value="female">Жена</option>
  </select>

  <label>
    <input type="checkbox" name="agree" required />
    Съгласен/а съм с условията
  </label>

  <label for="message">Съобщение:</label>
  <textarea
    id="message"
    name="message"
    rows="4"
    placeholder="Напиши нещо..."
  ></textarea>

  <button type="submit">Изпрати</button>
</form>
```

### Най-важни типове `<input>`:

| `type`     | Описание             |
| ---------- | -------------------- |
| `text`     | Текстово поле        |
| `email`    | Имейл (с валидация)  |
| `password` | Парола (скрит текст) |
| `number`   | Числово поле         |
| `date`     | Избор на дата        |
| `checkbox` | Отметка (да/не)      |
| `radio`    | Избор от група опции |
| `file`     | Качване на файл      |
| `submit`   | Бутон за изпращане   |

## 8. Семантични елементи (HTML5)

Семантичните елементи дават **смисъл** на структурата (вместо безлични `<div>`):

```html
<header>
  <nav>
    <a href="/">Начало</a>
    <a href="/about">За нас</a>
    <a href="/contact">Контакти</a>
  </nav>
</header>

<main>
  <article>
    <h2>Заглавие на статия</h2>
    <p>Съдържание на статията...</p>
  </article>

  <aside>
    <h3>Свързани теми</h3>
    <p>Допълнителна информация...</p>
  </aside>
</main>

<footer>
  <p>&copy; 2025 Моят сайт. Всички права запазени.</p>
</footer>
```

| Елемент     | Предназначение                          |
| ----------- | --------------------------------------- |
| `<header>`  | Горна секция (лого, навигация)          |
| `<nav>`     | Навигационни връзки                     |
| `<main>`    | Основно съдържание (едно на страница)   |
| `<article>` | Самостоятелно съдържание (статия, пост) |
| `<section>` | Тематична секция                        |
| `<aside>`   | Странично съдържание (sidebar)          |
| `<footer>`  | Долна секция (copyright, контакти)      |

> **Защо е важно?** Семантичният HTML подобрява SEO (търсачките разбират структурата), достъпността (screen readers) и четимостта на кода.

## 9. Мултимедия

### Видео

```html
<video width="640" controls>
  <source src="video.mp4" type="video/mp4" />
  Браузърът не поддържа видео.
</video>
```

### Аудио

```html
<audio controls>
  <source src="song.mp3" type="audio/mpeg" />
  Браузърът не поддържа аудио.
</audio>
```

### Вграждане (YouTube, Google Maps и др.)

```html
<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  allowfullscreen
>
</iframe>
```

## Практически задачи

### Задача 1 — Лична визитка

Създай HTML страница `vizitka.html`, която съдържа:

- Заглавие с твоето име (`<h1>`)

Създай страница `recepta.html` за любимо ястие:

- Заглавие с името на ястието
- Изображение на ястието

- Неподреден списък с продуктите

Създай `registracia.html` с формуляр, който включва:

- Поле за име и фамилия

- Поле за парола

- Checkbox за съгласие с условията
- Бутон за регистрация
- Всички полета трябва да имат `label`, `placeholder` и `required`

### Задача 4 — Уеб портфолио (обобщаваща)

Създай мини-сайт от **една страница** (`portfolio.html`) с:

- `<header>` с навигация (котвени линкове към секции)
- Секция „За мен" с текст и снимка
- Секция „Умения" с таблица
- Секция „Проекти" с 3 карти (всяка с `<article>`, заглавие, описание и линк)
- `<footer>` с copyright и линкове към социални мрежи

## Обобщение — какво научихме

1. **Структура** — всеки HTML документ има `<!DOCTYPE>`, `<html>`, `<head>`, `<body>`
2. **Текст** — заглавия (`h1`–`h6`), параграфи (`p`), форматиране (`strong`, `em`)
3. **Списъци** — подредени (`ol`) и неподредени (`ul`), вложени списъци
4. **Връзки** — `<a href="">` с атрибути `target` и `id` за навигация
5. **Изображения** — `<img>` с задължителен `alt` атрибут
6. **Таблици** — `table > thead/tbody > tr > th/td`
7. **Формуляри** — `form`, `input`, `select`, `textarea`, `button`
8. **Семантика** — `header`, `nav`, `main`, `article`, `section`, `aside`, `footer`
9. **Мултимедия** — `video`, `audio`, `iframe`
10. **Добри практики** — валиден код, семантични тагове, достъпност (`alt`, `label`)
